# Product

## Register

product

## Users

InternAgentS is used by people working in a local AI agent workspace. They move between conversations, workspace files, skills, service settings, and generated artifacts while trying to complete practical research, writing, coding, and automation tasks.

## Product Purpose

InternAgentS provides a local workbench for running DeepAgent workflows through a browser UI. Success means users can understand the current workspace, start or resume conversations, attach or inspect files, configure services, and enable skills without losing task context.

## Brand Personality

Calm, capable, precise. The interface should feel like a trustworthy productivity tool: quiet enough for repeated use, clear enough for first-time setup, and confident without feeling decorative.

## Anti-references

Avoid marketing-page composition, oversized hero treatments, decorative cards, heavy shadows, saturated inactive states, gratuitous motion, and inconsistent controls. The product should not feel like a theme demo or a landing page.

## Design Principles

- Keep the task surface quiet: primary content and active states should stand out without visual noise.
- Preserve context: navigation, skill activation, and workspace switching should make the current target obvious.
- Prefer familiar product patterns: standard buttons, tabs, dialogs, lists, and search controls are strengths here.
- Use density deliberately: compact information is welcome when it supports scanning and repeated action.
- Make state visible: loading, selected, installed, enabled, disabled, and error states should be clear in both light and dark modes.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Support keyboard navigation, visible focus, reduced-motion preferences, readable text at mobile widths, and robust light/dark mode behavior.
